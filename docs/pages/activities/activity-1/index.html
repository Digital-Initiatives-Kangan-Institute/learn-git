Activity 1 - Edit, Commit, Repeat

🎯 Scope

In this activity you will practice forking, cloning, branching, staging, committing, pushing and pull requests.

💡 Scenario

You have been hired by KangaCafe as a software developer to update their website. They don’t like the current orange color scheme and want you to experiment with different color options to find one that works best.

Note

  • You must use VSCode with the LivePreview extension to view your changes to the website. Follow this this guide to install the extension.

  • It is recommended to use the cheat sheet while doing this activity.

📋 Instructions

Using what you have learned about Git and GitHub, complete the following actions:

  1. Fork the activity repository
    • https://github.com/Digital-Initiatives-Kangan-Institute/git-activity-tbd2
  2. Clone your forked copy of the repository
  3. Create a new branch on your local copy of the repository
  4. Switch to the new branch you just created
  5. Open the index.html file
  6. Locate and change --color-scheme-primary from orange to dodgerblue
  7. Locate and change --color-scheme-secondary from darkorange to skyblue
  8. Stage, commit, and push your changes to GitHub
  9. Locate and change --color-scheme-primary from dodgerblue to purple
  10. Locate and change --color-scheme-secondary from skyblue to darkorchid
  11. Stage, commit, and push your changes to GitHub
  12. Locate and change --color-scheme-primary from purple to limegreen
  13. Locate and change --color-scheme-secondary from darkorchid to lime
  14. Stage, commit, and push your changes to GitHub
  15. Locate and change --color-scheme-primary and --color-scheme-secondary colors to the best fitting color scheme
  16. Stage, commit, and push your changes to GitHub
  17. Open a pull request, merging the changes from your new branch into the master branch
